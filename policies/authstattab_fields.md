# authstattab — Authorization Status Table — Field Reference

The authorization status table holds one row per authorization attempt. It is
populated at auth-time by the switch. Use it for anything involving approval /
decline decisions, the merchant at the moment of the transaction, response
codes, and issuer/acquirer routing.

## Columns

- **acctnum** — Account number / PAN (primary account number). String, leading zeros are significant.
- **txndate** — Transaction date (the date on which the auth request was made).
- **txntime** — Transaction time (time-of-day of the auth request).
- **issuer** — Issuer IIC (Interbank Identification Code of the card issuer).
- **acquirer** — Acquirer IIC (Interbank Identification Code of the acquirer bank that submitted the auth).
- **tracenum** — System Trace Audit Number. Generated when the auth is initiated; correlates messages across systems during a single auth cycle.
- **req_amount** — Requested amount in local currency. Treat as a string — do not cast to numeric for lookup filters.
- **curr_code** — Currency code of the requested amount (ISO 4217 numeric).
- **issuer_amt** — Issuer amount (amount as seen by the issuer, currency-converted if needed).
- **d_amt** — Dollar amount (USD equivalent).
- **approval_code** — Approval code issued by the issuer on successful authorization.
- **mcc** — Merchant category code. String, 4 digits, leading zeros significant (e.g. `0742` ≠ `742`).
- **action_code** — Response code generated by the issuer / network. Used to determine whether the transaction was **approved** or **declined**. See business rules for the mapping.
- **funccode** — Function code. Describes the nature of the transaction (e.g. authorization, reversal, pre-auth completion).
- **add_resp** — Additional response. Populated by issuer / network based on CAVV, CVV, AVS, and similar validations. Helps explain a decline reason.
- **response_time** — Time taken (in ms) to complete the auth cycle end-to-end.
- **swoutind** — Switch-out indicator. Identifies who took the decision for this transaction — the network or the issuer.
- **posid** — POS ID. Each element of the POS ID helps identify the transaction type (e.g. card-present, card-not-present, terminal capability).
- **forwaiic** — Forwarding IIC. The next-hop institution that forwarded the message.
- **merchant_num** — Merchant number assigned to the establishment.
- **mrch_nm** — Merchant name.
- **mrch_str_adr** — Merchant street address.
- **mrch_cty_nm** — Merchant city name.
- **merchant_geo_cde** — Merchant geo code (country/region code).
- **region** — High-level regional bucket derived from the merchant's geography. One of: **APAC** (Asia-Pacific), **EMEA** (Europe, Middle East, Africa), **AMER** (Americas). Use for broad regional splits (e.g., "decline rate by region").
- **auth_type_code** — Authorization type code. Used to validate whether the transaction is **PB secured** or not. See business rules for the mapping.
- **nrid** — Network Reference ID. A unique identifier per transaction — useful for cross-system joins where `tracenum` is ambiguous.
- **mti** — Message Type Identifier (ISO 8583 MTI, e.g. `0100`, `0110`, `0200`, `0210`).
